# Todo Full Stack Application

## Short Description

This is a full stack Todo application where users can create accounts, log in, and manage their todo items. Users can create, update, delete, and mark todos as completed. The application includes authentication features like signup, login, password reset, and Google OAuth login.

## Technologies Used

- React + TypeScript
- Node.js + TypeScript
- MongoDB
- Zustand
- React Query
- React Hook Form
- JWT Authentication

## Features

### User Authentication
- User Signup: Create a new account with email and password
- User Login: Login with email and password
- Forgot and Reset Password: Users can request a password reset link via email. Note: Email verification works, but Phone OTP verification will not work because the free limit is completed.
- Google OAuth: Login and signup using Google account

### Todo Management
- Todo CRUD Operations:
  - Create: Add new todo items
  - Read: View all todos
  - Update: Edit existing todos
  - Delete: Remove todos
- Mark Todo as Completed or Not Completed: Toggle completion status of todos

## How to Run

### Prerequisites
- Node.js installed on your system
- MongoDB Atlas account (free tier works)
- Git (optional, for cloning)

### Step 1: Install Dependencies

#### Frontend
```bash
cd client
npm install
```

#### Backend
```bash
cd server
npm install
```

### Step 2: Set Up Environment Variables

#### Backend (.env file in server/ directory)
Create a `.env` file in the `server/` directory with the following variables:

```
PORT=3000
MONGO_URI=your_mongodb_atlas_connection_string
JWT_KEY=your_secret_jwt_key
NODE_ENV=development

# Email Configuration (for password reset)
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password

# Google OAuth (optional, for Google login)
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=postmessage

# Frontend URL (for CORS)
FRONTEND_URL=http://localhost:5173
```

#### Frontend (.env file in client/ directory)
Create a `.env` file in the `client/` directory with the following variables:

```
VITE_API_URL=http://localhost:3000/api
VITE_GOOGLE_CLIENT_ID=your_google_client_id
```

### Step 3: Start the Servers

#### Start Backend Server
```bash
cd server
npm run dev
```
The backend server will run on `http://localhost:3000`

#### Start Frontend Server
Open a new terminal window:
```bash
cd client
npm run dev
```
The frontend application will run on `http://localhost:5173`

### Step 4: Access the Application
Open your browser and go to `http://localhost:5173`

## Assumptions

The UI is kept simple because the main focus was on implementing functionality rather than complex design. The application prioritizes working features over advanced styling.

## Error Handling

All errors that occur in the application are automatically logged into MongoDB in a separate collection. This helps in debugging and monitoring the application. Error logs include details like error message, timestamp, and the route where the error occurred.

## Database

The application uses a free MongoDB Atlas instance. MongoDB Atlas provides a free tier that is sufficient for development and small projects. The database stores user information, todos, and error logs.
